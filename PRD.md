# PRD (Product Requirements Document) - 北科遊戲化學業交流區 (PetScholar)

## 1. 專案背景與定位
在大學校園中，跨學科的學術討論與同儕解答是學習的重要一環，但傳統討論區缺乏鼓勵學生積極
回答課業難題的誘因。**PetScholar** 結合**論壇討論**、**遊戲化虛擬寵物養成**與**即時共讀**：
學生在論壇解答課業問題獲得「金幣」與「經驗」，金幣可在商城購買食物餵食虛擬寵物、養成等級，
自習室則提供番茄鐘、目標與即時文字/語音/視訊共讀，建立「同儕互助、答題養成、一起讀書」的
社群氛圍。

> 本專案原為純靜態 Demo，現已實作為 **Next.js 全端應用 + 自訂 Socket.IO 伺服器**：真實
> PostgreSQL 資料庫、Google 登入、即時通訊、物件儲存與 WebRTC 語音/視訊，部署於 Zeabur。

## 2. 目標使用者與場景
- **求助學生**：遇到課業難題時於對應科系看板發文（可附圖、設懸賞）。
- **解答學霸**：解答問題，被採納時獲得金幣與經驗，照顧並升級自己的虛擬寵物。
- **共讀夥伴**：在自習室一起用番茄鐘專注、設定目標、即時聊天或語音/視訊共讀。
- **管理者（教授/系統管理員）**：審閱提問、處理檢舉、管理使用者與全站可管理資料。

## 3. 技術規格
- **前端/全端**：Next.js 16（App Router、React 19）、TypeScript、Tailwind v4（Material 3 token + 深淺模式）。
- **後端**：自訂 Node 伺服器（`server.mjs`）包 Next + **Socket.IO**（即時聊天、留言即時化、WebRTC 信令）。**PostgreSQL + Drizzle ORM**。**Auth.js v5 + Google 登入**（資料庫 session）。
- **即時/媒體**：**Redis**（Socket.IO adapter）、**MinIO/S3**（頭像/附圖/錄製）、**WebRTC mesh + Cloudflare TURN**（語音/視訊）、**RNNoise** AI 降噪、**KaTeX** 數學式。
- **部署**：Zeabur，啟動指令 `npm start`（自訂伺服器）。機密以環境變數注入，不入庫。

## 4. 功能需求 (Functional Requirements)

### F1. 跨學院看板與提問
- 6 學院看板（機電/電資/工程/管理/設計/人文社科），各看板顯示提問列表（標題、作者、懸賞、時間、狀態、標籤）。
- 發問支援標籤、懸賞（金幣託管）、附圖、KaTeX 數學式。首頁有熱門標籤與熱門提問。

### F2. 即時 Q&A 與解答採納
- 文章詳情頁以**樹狀結構 (Tree)** 渲染多層巢狀留言，並繪成 SVG Node-Link 圖（HTML 留言 ↔ 節點雙向高亮）。
- 留言**即時更新**（Socket.IO）：他人新增留言/採納會即時反映。輸入框支援 Enter 送出、Shift+Enter 換行、emoji、附圖、IME 安全。
- **採納**：發問者（或助教認證）採納解答 → 託管懸賞 + 基礎獎勵(+15) + 升級獎勵發給解答者，發問者亦得小額獎勵，文章標記「已解決」。

### F3. 虛擬寵物、金幣經濟與等級制度
- 寵物狀態：HP（愛心）、EXP、等級、頭銜（學習新芽→學術大師）、金幣。左側欄全站一致顯示，含每日簽到與治療。
- **賺金幣**：採納（基礎+懸賞）、被採納、每日簽到、升級獎勵。
- **商城**：購買食物/配件。**等級解鎖**（部分商品需達指定寵物等級）。餵食恢復 HP 與 EXP，升級提升上限並發獎勵+慶祝提示。

### F4. 自習室即時共讀
- 番茄鐘（持久化、切走不重置）、讀書目標（新增/勾選/刪除）。
- **即時文字聊天**（Socket.IO + DB 持久化）。
- **WebRTC 語音/視訊通話**：mesh + Cloudflare TURN。**強制錄製**（音訊一律錄、開鏡頭連影像錄）且全程顯示「●錄音/錄影中」與加入前同意提示（隱私告知）。Discord 風 speaking 綠光環。RNNoise AI 降噪。
- **房間管理**：建立者可編輯房間資訊、指定房間管理員、禁麥/禁鏡/踢人、設密碼。列表整卡可進入、顯示創建者。

### F5. 科系、公開檔案與頭像
- 科系清單由管理員維護。所有「選科系」處皆下拉限清單。
- 公開個人檔案 `/u/[id]`（點頭像進入）顯示系所/寵物/等級/統計。
- 使用者可上傳自訂頭像（MinIO）。

### F6. 排行榜、成就與後台
- 排行榜：探索榜（留言）、學科榜（貼文）、自習參與榜（皆以真實資料聚合）。個人成就徽章、福利社券。
- 教授後台、系統管理後台（依角色控管）：可管理看板/貼文/留言/自習室/商城/使用者(全資訊與角色)/檢舉/聊天訊息/語音錄影/科系。

### F7. 資料結構視覺化
- 以 SVG 留言樹狀圖示範 **Tree**。看板/科系以 **Object/Map** 查找。文章/留言/商品/排行榜以 **Array** 遍歷。

## 5. 非功能需求 (Non-Functional Requirements)
- **安全**：所有資料異動於 server 端驗身分與權限。media file route 防路徑穿越、拒 SVG 上傳、`nosniff`/CSP。安全標頭（HSTS、X-Frame-Options 等），並允許自身來源使用麥克風/鏡頭。
- **隱私**：語音/視訊強制錄製前明確告知並取得同意，通話中全程顯示錄製指示。
- **一致性**：全站沿用設計 token 與深淺模式。使用者資料變更會 revalidate 受影響頁面。
- **可用性**：空狀態給明確引導。RWD 支援手機。保留「3 分鐘簡報導覽」引導模式。
